## 1. Description

This Skill inspects text for regulatory citations (FDA 510(k) numbers, FDA guidance document titles, Code of Federal Regulations sections) and decides whether each citation is verifiable against a primary source.

**The core mandate: never let a hallucinated citation reach the user.**

In regulated domains, a fabricated citation is not a minor error ŌĆö it is a reputational and potentially legal failure. This Skill exists so the rest of SafeSignal can confidently emit regulator-facing output without fear of invented references.

---

## 2. When to Use

### Activate this Skill when:
- Any text generated by another Skill or module contains what appears to be a regulatory citation
- Preparing output for a user-facing surface (dashboard, PSUR report, CLI response)
- Testing another Skill's output for hallucination rate
- Running the pre-output gate mandated by FR-24

### Do NOT activate this Skill when:
- Citing MAUDE report IDs (different system ŌĆö use `maude-narrative-extractor`'s validation)
- Citing internal SafeSignal document sections (e.g., "Doc 3 ┬¦5") ŌĆö these are self-references
- The input is pure data (no textual claims)
- Generating citations from scratch ŌĆö this Skill verifies existing ones, it does not invent them

## 3. Inputs

### Required Input
```

  "text_to_verify": "string - the output text containing citations",
```

### Citation Types This Skill Recognizes

| Type | Pattern | Verification Source |
|---|---|---|
| 510(k) K-number | `K\d{6}` | openFDA `/device/510k.json` API |
| De Novo | `DEN\d{6}` | openFDA `/device/classification.json` |
| PMA | `P\d{6}` | openFDA `/device/pma.json` |
| CFR section | `21\s?CFR\s?(Part\s?)?\d+(\.\d+)?` | Curated regex + known sections list |
| FDA guidance title | "..." in quotes, next to "FDA guidance" | Local FDA guidance index file |
| FDA warning letter | `WARNING LETTER` with date | Not verified in v1.0 ŌĆö flagged for review |

## 4. Outputs

### Output Schema (see `schemas/output.schema.json`)
```json
{
  "verification_ts": "ISO-8601 timestamp",
  "skill_version": "1.0.0",
  "citations_found": [
    {
      "citation_type": "k_number | de_novo | pma | cfr | guidance_title | unknown",
      "raw_text": "exact substring as it appeared",
      "start_char": "integer offset",
      "end_char": "integer offset",
      "verified": "true | false",
      "verification_source": "openfda_510k | openfda_pma | cfr_regex | guidance_index | none",
      "verification_details": "optional string (e.g., device name for K-number)",
      "action": "keep | flag | redact"
    }
  ],
  "overall_verdict": "clean | contains_unverified | contains_fabricated",
  "safe_to_emit": "boolean",
  "redacted_text": "string ŌĆö input text with unverified citations replaced by [UNVERIFIED CITATION]"
}
```

### Verdict Rules
- `clean` ŌĆö All citations verified OR no citations found
- `contains_unverified` ŌĆö Some citations couldn't be verified (e.g., guidance index stale)
- `contains_fabricated` ŌĆö At least one citation is demonstrably fake (e.g., K-number not in openFDA)

### The Safety Rule
- If `overall_verdict == "contains_fabricated"` ŌåÆ `safe_to_emit = false`
- If `strict_mode == true` and `overall_verdict == "contains_unverified"` ŌåÆ `safe_to_emit = false`
- Otherwise ŌåÆ `safe_to_emit = true`

## 5. Procedure

1. **Scan** the input `text_to_verify` for citation patterns (all types in ┬¦3 table). Record each hit with exact character offsets.

2. **For each K-number hit:**
   - Query openFDA `/device/510k.json?search=k_number:"KXXXXXX"`
   - If API returns Ōēź1 result ŌåÆ `verified=true`, store device name in `verification_details`
   - If API returns 0 results ŌåÆ `verified=false`, `action=redact`
   - If API error ŌåÆ `verified=false`, `verification_source="none"`, `action=flag`

3. **For each De Novo / PMA hit:** same process against their respective openFDA endpoints.

4. **For each CFR hit:**
   - Validate against regex: `^21\s?CFR\s?(Part\s?)?(\d{1,3})(\.\d{1,4})?$`
   - Cross-check the Part number against the list of valid medical-device CFR Parts: `800ŌĆō899`
   - If Part number outside that range ŌåÆ `verified=false`, `action=flag`

5. **For each guidance title hit:**
   - Look up the exact title in the local FDA guidance index (`data/fda_guidance_index.json`)
   - Fuzzy-match allowed: Levenshtein distance Ōēż2 on normalized (lowercase, punctuation-stripped) strings
   - If no match ŌåÆ `verified=false`, `action=flag`

6. **Construct `redacted_text`:**
   - For every citation with `action=redact`, replace the raw text with `[UNVERIFIED CITATION: <citation_type>]`
   - For every citation with `action=flag`, leave in place but add a footnote
   - For every citation with `action=keep`, do nothing

7. **Compute `overall_verdict`:**
   - Any `action=redact` ŌåÆ `contains_fabricated`
   - Any `action=flag` but no `redact` ŌåÆ `contains_unverified`
   - None of above ŌåÆ `clean`

8. **Set `safe_to_emit`** per ┬¦4 safety rule.

9. **Return** the full output object.

## 6. Rules & Constraints

### NEVER
- NEVER invent a device name, manufacturer, or detail to "fill in" a verification result.
- NEVER return `verified=true` without a successful primary-source lookup.
- NEVER silently omit a citation from the output ŌĆö every detected citation is reported.
- NEVER modify the raw source text in ways the user cannot reverse or inspect.
- NEVER use Claude's training-data knowledge to "verify" a citation. The training data is not a primary source.

### ALWAYS
- ALWAYS return the exact character offsets for every citation detected.
- ALWAYS log every API lookup to the LLM audit trail (even non-LLM lookups) for ALCOA+ compliance.
- ALWAYS prefer false-negative over false-positive: if uncertain, flag rather than pass.
- ALWAYS record the `verification_source` so a human can re-run the check.
- ALWAYS include `skill_version` in every output for auditability (FR-12).

### EDGE CASES
- **Empty input text** ŌåÆ Return `{citations_found: [], overall_verdict: "clean", safe_to_emit: true}`
- **Malformed citation** (e.g., `K12`) ŌåÆ Pattern doesn't match, ignored. Not a bug.
- **Ambiguous text** (e.g., "K123456 or possibly K123457") ŌåÆ Treat each as separate citation.
- **API timeout** ŌåÆ `verified=false`, `action=flag`, `verification_source="none"`. Do not retry inside this Skill; that's the caller's job.
- **Conflicting sources** (e.g., openFDA says yes, guidance index says no) ŌåÆ Trust openFDA for K-numbers and PMAs; trust the guidance index for guidance titles. Document disagreement in `verification_details`.


## 8. Validation

This Skill is validated by the test set at `tests/fixtures/citation_verifier_test_set.jsonl`, which includes:

- 20 inputs containing verified real citations (expect all `verified=true`)

### Passing criteria (per FR-24):
- 100% of known-fake citations flagged or redacted
- 100% of known-real citations pass as verified
- Zero `verified=true` without a corresponding `verification_source` ŌēĀ `"none"`
